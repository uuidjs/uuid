Random # generation is vital for avoiding uuid collisions, but there's no silver bullet option since Math.random isn't guaranteed to be "cryptographic quality". So we feature-detect based on API availability.

(Each RNG API is normalized here to return 128-bits (16 bytes) of random data)

 
  var mathRNG, nodeRNG, whatwgRNG;
¶

Math.random-based RNG. All browsers, fast (~1M/sec), but not guaranteed to be cryptographic quality.

 
  var _rndBytes = new Array(16);
  mathRNG = function() {
    var r, b = _rndBytes, i = 0;

    for (var i = 0, r; i < 16; i++) {
      if ((i & 0x03) == 0) r = Math.random() * 0x100000000;
      b[i] = r >>> ((i & 0x03) << 3) & 0xff;
    }

    return b;
  }
